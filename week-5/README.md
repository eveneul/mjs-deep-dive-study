# <실행 컨텍스트>

실행 컨텍스트란 **코드가 실행되는 환경**을 뜻한다. 코드가 실행되려면 **소스코드 평가**와 **소스코드 실행**이라는 두 가지 단계를 거치게 되는데, 이 과정에서 필요한 정보들을 모아둔 실행단위다.
즉, 실행 컨텍스트는 컨텍스트 제어를 하고, 코드의 실행 순서를 제어한다.

## 1. 소스코드의 4가지 타입

ECMAScript 사양은 소스코드를 4가지 타입으로 구분하며, 이 4가지 타입의 소스코드는 **실행 컨텍스트를 생성**한다.

### 1) 전역 코드

- 전역 변수를 관리하기 위해 전역 스코프를 생성하고, 전역 코드가 평가되면 전역 실행 컨텍스트를 생성한다.
- `var`로 선언된 전역 변수와 함수 선언문으로 정의된 전역 함수를 전역 객체의 프로퍼티와 메서드로 바인딩한다.

### 2) 함수 코드

- 함수가 호출될 때마다 함수 실행 컨텍스트를 새로 만든다.
- 지역 스코프를 생성하고, 지역 변수, 매개변수, arguments 객체를 관리한다.
- `this` 바인딩도 이 시점에 결정된다.

### 3) `eval` 코드

- `strict mode`에서 독자적인 스코프를 갖는 실행 컨텍스트를 생성한다.

### 4) 모듈 코드

- 모듈별로 독자적인 스코프를 갖는 실행 컨텍스트를 생성한다.
- `<script type="module">`이나 ES 모듈에서 동작한다.

## 2. 소스코드의 평가와 실행

- JS 엔진은 소스코드를 한 번에 실행하는 것이 아닌 평가와 실행을 나눠서 처리한다.
- 평가 단계
  - **실행 컨텍스트를 생성**하고, 변수·함수 선언문을 먼저 읽어 실행 컨텍스트에 등록한다.
  - 이것이 **호이스팅**이다.
  - `var` 변수나 함수 선언식이 선언 전에도 참조가 가능한 것처럼 보이게 하는 것도 이 이유이다.
- 실행 단계
  - 평가 단계에 만들어진 실행 컨텍스트를 바탕으로 코드를 순차적으로 실행한다.
  - 변수에 값을 할당하거나, 함수를 호출하는 것이 이 단계에서 이루어진다.
  - 값이 필요하면 실행 컨텍스트에서 **식별자를 검색하여 가져온다.**

## 3. 실행 컨텍스트 스택 (= 콜 스택)

- JS 엔진은 실행 컨텍스트를 **스택(stack) 자료구조**로 관리한다.
- 이것을 실행 컨텍스트 스택 또는 콜 스택(Call stack)이라고 부른다.
- 콜 스택 구조 덕분에 JS가 실행 스레드이면서도 코드 실행 순서를 명확하게 관리할 수 있다.
- 스택이 비어 있으면 더 이상 실행할 코드가 없다는 뜻이고,
- 이벤트 루프가 태스크 큐에서 새 작업을 가져와 스택에 올리는 방식으로 **비동기가 동작한다.**

### 실행 컨텍스트 스택 순서

- 전역 코드가 평가되면 전역 실행 컨텍스트가 스택에 push된다.
- 이후 함수가 호출되면 함수 실행 컨텍스트가 스택에 push된다.
- 함수가 종료되면 pop되면서 직전 컨텍스트로 돌아온다.
- 스택의 최상단에 있는 컨텍스트가 **현재 실행 컨텍스트**다.

## 4. 랙시컬 환경

- 실행 컨텍스트 내부에서 실질적으로 식별자와 값을 관리하는 자료구조다.
- 실행 컨텍스트는 `LexicalEnvironment`와 `VariableEnvironment` 두 가지의 컴포넌트로 구성된다.
  - 처음에는 같은 렉시컬 환경을 참조하다가, 실행되면서 분리될 수 있다.
  - `let`, `const`, 함수 선언은 `LexicalEnvironment`
  - `var`는 `VariableEnvironment`에 등록된다.
- 환경 레코드와 외부 랙시컬 환경에 대한 참조라는 두 가지의 컴포넌트로 구성된다.

## 5. 실행 컨텍스트의 생성과 식별자 검색 과정

```js
var x = 1;
const y = 2;

function foo(a) {
  var x = 3;
  const y = 4;

  function bar(b) {
    const z = 5;
    console.log(a, b, x, y, z);
  }

  bar(10);
}

foo(20);
```

### 1) 전역 객체 생성

- 실행 컨텍스트가 실행되기도 전에, JS 엔진은 먼저 전역 객체를 생성한다.
  - 브라우저라면 `window`, Node.js 환경이면 `global`에 생성된다.
- 전역 객체에는 빌트인 전역 프로퍼티, 빌트인 전역 함수, 빌틓인 객체, 그리고 브라우저라면 Web API가 미리 담겨 있다.
- 전역 객체는 어떤 객체에도 속하지 않는 최상위 객체이고, 이 시점에는 코드가 실행되기 전인 상태이다.

### 2) 전역 코드 평가

- 전역 코드 평가가 실행되면 **전역 실행 컨텍스트**가 생성되고, 콜 스택에 Push된다.
- 이 컨텍스트 안에 **전역 랙시컬 환경**이 만들어지고, 실행 컨텍스트와 연결된다.
- 전역 렉시컬 환경은 두 개의 환경 레코드로 구성된다.
  - **객체 환경 레코드(Object Environment Record)**
    - `var` 키워드로 선언한 전역 변수와 함수 선언문으로 정의한 전역 함수, 빌트인 전역 프로퍼티와 빌트인 전역 함수, 표준 빌트인 객체를 관리한다.
    - `BindingObject`라는 속성이 전역 객체를 가리키고, 식별자를 전역 객체 프로퍼티로 추가하는 방식으로 동작한다.
    - `var x = 1`로 선언하면 `window.x`로 접근할 수 있는 이유이기도 하다.
    - 이 시점에서 `x`는 `undefined`로 초기화된다. (호이스팅)
  - **선언적 환경 레코드(Declarative Environment Record)**
    - `let`, `const`로 선언된 변수를 관리하고, 전역 객체와 무관하게 존재한다.
    - `const y = 2`으로 선언하면 `window.y`로 접근할 수 없는 이유다.
- 전역 `this` 바인딩은 전역 객체(`window`)를 가리킨다.
- 전역 렉시컬 환경의 외부 렉시컬 환경 참조는 `Null`이다. 전역이 스코프 체인의 최상단이기 때문이다.

### 3) 전역 코드 실행

- 평가가 끝나면 전역 코드가 순차적으로 실행된다.
- `var x = 1`이 실행되면서 객체 환경 레코드에 이미 `undefined`로 등록된 변수가 `1`로 재할당된다.
- `const y = 2`가 실행되면서 선언적 환경 레코드의 `y`가 `<uninitialized>`에서 `2`로 바뀐다. -> 이 시점에서 TDZ이 해재된다.
- `foo(20)`에서는 함수를 호출하기 전에 **식별자 검색**이 일어난다.
- 현재 실행 중인 컨텍스트의 환경 레코드에서 `foo`를 찾는다. 객체 환경 레코드에 `foo`가 있고, 값은 함수 객체이므로 호출을 진행한다.

### 4) `foo` 함수 실행 컨텍스트 생성 (= `foo` 함수 평가)

- `foo(20)`이 실행되면 `foo` 함수 실행 컨텍스트가 생성되고, 콜 스택 최상단에 push된다.
- foo의 렉시컬 환경 안에 있는 함수 환경 레코드에는 `arguments` 객체, 매개변수 `a`, `var x`, 함수 선언문 `bar`가 등록된다.

### 5) `foo` 함수 코드 실행

- `foo` 실행 단계에서 `var x = 3`가 실행되어 `x`가 `3`으로 (재)할당되고, `const y = 4`가 실행되어 `y`가 `4`로 초기화된다.
- `bar(10)`을 만나면 다시 식별자를 검색한다. 현재 실행 컨텍스트인 `foo`의 환경 레코드에서 `bar`를 찾고, 함수 환경 레코드에 `bar`가 있으므로 호출을 진행한다.

### 5) `bar` 함수 실행 컨텍스트 및 함수 코드 실행

### 6) 실행 컨텍스트 복귀

- `bar` 실행이 끝나면 bar 실행 컨텍스트가 콜 스택에서 pop되고, 현재 실행 중인 실행 컨텍스트는 `foo`로 돌아온다.
- `foo`도 실행이 끝나면 실행 컨텍스트가 콜 스택에서 pop되고, 전역 실행 컨텍스트로 돌아온다.
- 전역 코드가 모두 끝나면 전역 컨텍스트도 pop되며 콜 스택이 빈다.

```
전역 객체 생성
  → 전역 실행 컨텍스트 생성 & 전역 렉시컬 환경 구성
    → 전역 코드 실행 (x=1, y=2 할당)
      → foo 호출 → foo 실행 컨텍스트 push
        → foo 코드 실행 (x=3, y=4 할당)
          → bar 호출 → bar 실행 컨텍스트 push
            → console.log 실행 (스코프 체인 타고 식별자 검색)
          → bar 컨텍스트 pop
        → foo로 복귀
      → foo 컨텍스트 pop
    → 전역으로 복귀
  → 전역 컨텍스트 pop → 콜 스택 비움
```
