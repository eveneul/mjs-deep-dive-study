# 변수

## 변수 선언과 메모리 영역

```jsx
var a; // 선언, undefined으로 할당

a = "abc"; // 값 할당
```

변수: **변할 수 있는 데이터**, 이 데이터의 식별자는 a로 한다.

변수 a를 선언하면 초기에는 `undefined` 로 초기화가 된다. 뒤이어 할당까지 하면 그 변수는 `abc` 라는 값이 된다. 사실 값이 되었다고 하지만 **변수 선언과 할당은 메모리 공간에 값을 넣고, 그 값이 있는 주소를 참조하는 식이다.**

- 변수 영역

| 주소      | …   | 1002 | 1003    | 1004 | …   |
| --------- | --- | ---- | ------- | ---- | --- |
| 데이터    |     |      | 이름: a |
| 값: @5004 |     |      |

- 데이터 영역

| 주소   | …   | 5002 | 5003 | 5004  | …   |
| ------ | --- | ---- | ---- | ----- | --- |
| 데이터 |     |      |      | ‘abc’ |     |

1. 변수 영역에서 빈 공간 (@1003)을 확보한다
2. 확보한 공간의 식별자를 a로 지정한다
3. 데이터 영역의 빈 공간(@5004)에 문자열 ‘abc’를 지정한다.
4. 변수 영역에서 a라는 식별자를 검색한다 (@1003)
5. 앞서 저장한 문자열의 주소(@5004)를 @1003의 공간에 대입한다.

**💡 변수 영역에 값을 직접적으로 대입하지 않고 굳이 한 단계를 거치는 이유**
-> 데이터 변환을 더 자유롭게 할 수 있게 함과 동시에 메모리늘 더욱 효율적으로 관리하기 위해

만약 `a` 라는 변수의 값이 ‘abc’가 아닌 123으로 변경되었으면 데이터 영역에서 @5004의 값이 123으로 덮어 씌워지는 것이 아니라 다른 메모리 영역에 123이라는 값이 담기고, 변수 메모리 영역에서 값이 @5004가 아닌 123이라는 값이 있는 주소로 변경된다.

# 데이터 타입

| number    | 정수, 실수 구분 없이 하나의 숫자 타입. 64비트 부동소수점            |
| --------- | ------------------------------------------------------------------- |
| string    | 문자열, 불변값                                                      |
| boolean   | true / false                                                        |
| undefined | 선언은 되었지만 값이 할당되지 않은 상태 (엔진이 자동으로 할당해 줌) |
| null      | 값이 없다는 것을 **의도적으로** 명시 (개발자가 할당)                |
| symbol    | ES6에서 추가된 타입. 유일무이한 값                                  |
| bigint    | ES2020에서 추가된 타입. 아주 큰 정수                                |

## 객체 타입

원시 타입을 제외한 모든 것(객체, 배열, 함수)이 여기에 해당.

## 숫자 타입의 특이점

```jsx
console.log(0.1 + 0.2); // 0.30000000000000004
// 부동소수점 오차! 정밀한 계산엔 주의 필요

console.log(1 / 0); // Infinity
console.log(-1 / 0); // -Infinity
console.log(1 * "a"); // NaN (Not a Number)
```

## `undefined` vs `null`

```jsx
var x;
console.log(x); // undefined — 엔진이 자동으로 할당한 초기화가 안 됨 상태

var y = null;
console.log(y); // null — 개발자가 이 변수에 값 없다고 명시한 것
```

## 동적 타이핑

자바스크립트는 변수에 타입이 없고 **값에 타입이 있다.** 그래서 같은 변수에 다른 타입의 값을 재할당 할 수 있다.

유연하지만 예측하기 어려운 버그를 야기하기도 한다. Typescript가 나온 이유 중 하나다.

# 연산자

## 산술 연산자

```jsx
5 + 2; // 7
5 - 2; // 3
5 * 2; // 10
5 / 2; // 2.5
5 % 2; // 1 (나머지)
5 ** 2; // 25 (거듭제곱, ES7)
```

단항 산술 연산자로 `++` , `--` 가 있고, **위치에 따라 동작이 다르다.**

```jsx
var x = 1;
var a = x++; // a = 1, x = 2 (후위: 현재 값 반환 후 증가)
var b = ++x; // b = 3, x = 3 (전위: 먼저 증가 후 값 반환)
```

## 비교연산자

```jsx
// == 동등 비교: 타입 변환(암묵적 강제 변환) 후 비교
5 == "5"; // true
0 == false; // true
null == undefined; // true

// === 일치 비교: 타입과 값 모두 비교
5 === "5"; // false
0 === false; // false
```

`==` 는 타입이 다르면 강제로 변환해서 비교하기 때문에 예측이 어려우므로 웬만해서는 `===` 로 타입까지 비교하는 연산자를 사용하는 것이 좋다.

## 논리 연산자와 단락 평가

```jsx
// && : 왼쪽이 falsy면 왼쪽 값 반환, 아니면 오른쪽 값 반환
true && "hello"; // 'hello'
false && "hello"; // false
null && "hello"; // null

// || : 왼쪽이 truthy면 왼쪽 값 반환, 아니면 오른쪽 값 반환
false || "hello"; // 'hello'
"world" || "hello"; // 'world'
```

## typeof 연산자

```jsx
typeof 1; // 'number'
typeof "hello"; // 'string'
typeof true; // 'boolean'
typeof undefined; // 'undefined'
typeof null; // 'object' ← 유명한 버그! null인데 object가 나옴
typeof {}; // 'object'
typeof []; // 'object'
typeof function () {}; // 'function'
```

`typeof null === 'object'` 는 자바스크립트의 초기 버그다. 하지만 하위 호한성 때문에 수정되지 않았으니, `null` 체크는 `=== null` 로 해야 한다.

# 제어문

## 조건문

- `if else`

  ```jsx
  if (조건식) {
    // 참일 경우 실행
  } else {
    // 거짓일 경우 실행
  }
  ```

  - 조건식을 추가하여 조건에 실행될 코드를 늘리고 싶으면 `else if` 를 추가하면 된다.

- `switch`

  ```jsx
  switch (day) {
    case 1:
      console.log("월");
      break; // break가 없으면 다음 case로 폴스루(fall-through)
    case 2:
      console.log("화");
      break;
  }
  ```

  - 값이 일치하는 case로 이동한다.

## 반복문

```jsx
// for
for (let i = 0; i < 5; i++) { ... }

// while
let i = 0;
while (i < 5) { i++; }

// do-while: 조건과 상관없이 최소 1번은 실행
do { i++; } while (i < 5);
```

`break` / `continue`

- `break` → 반복문 또는 `switch` 문을 즉시 탈출
- `continue` → 현재 반복을 건너뛰고 다음 반복으로 이동

# 타입 변환과 단축 평가

## 타입 변환

### 명시적 타입 변환 (= 타입 캐스팅)

개발자가 의도적으로 값의 타입을 변환하는 것을 **명시적 타입 변환** 또는 **타입 캐스팅**이라고 한다.

```jsx
// 숫자 → 문자열
String(1)(
  // '1'
  1,
).toString(); // '1'
1 + ""; // '1' ← 이건 암묵적

// 문자열 → 숫자
Number("1"); // 1
parseInt("1") + // 1
  "1"; // 1 ← 단항 + 연산자
```

### 암묵적 타입 변환 (= 타입 강제 변환)

자바스크립트가 자동으로 타입을 변환하는 것. 의도하지 않은 동작과 오류의 원인이 될 수도 있다.

```jsx
1 + "2"; // '12' — 숫자가 문자열로 변환
"5" - 1; // 4   — 문자열이 숫자로 변환
"5" * 2; // 10
!!0; // false — Falsy값
!!"hello"; // true  — Truthy값
```

## 단축 평가

### 논리 연산자를 이용한 단축 평가

- 논리곱(`&&`): 두 개의 피연산자가 모두 `true` 로 평가될 때 `true` 를 반환한다. 논리곱 연산자는 좌항에서 우항으로 평가가 진행된다.
- 논리합(`||` ): 두 개의 피연산자 중 하나만 `true` 여도 `true` 를 반환한다.

## Falsy 값

boolean 변환 시 `false` 가 되는 값

```jsx
(false, 0, -0, 0n, "", null, undefined, NaN);
// 이 8가지를 제외한 모든 값은 Truthy
```

## 옵셔널 체이닝 `?.` 과 null 병합 연산자 `??`

- 옵셔널 체이닝(`?.` ): 좌항의 피연산자가 `null` 또는 `undefined` 인 경우 `undefined` 를 반환하고, 그렇지 않으면 우항의 프로퍼티를 참조한다.
  ```jsx
  // ?. : 왼쪽이 null/undefined면 undefined 반환 (에러 안 남)
  user?.address?.city;
  ```
- null 병합 연산자(`??` ): 좌항의 피연산자가 `null` 또는 `undefined` 인 경우 우항의 피연산자를 반환하고, 그렇지 않으면 좌항의 피연산자를 반환한다.
  ```jsx
  // ?? : 왼쪽이 null/undefined일 때만 오른쪽 반환 (|| 와 다름)
  const name = user.name ?? "익명";
  // || 는 falsy(0, '' 등)에도 반응하지만, ?? 는 null/undefined에만 반응
  ```

# 객체

자바스크립트는 **객체 기반 언어**이고, 원시 타입을 제외한 모든 것이 객체다. 원시 타입도 필요 시 래퍼 객체로 감싸진다.

객체는 **프로퍼티**의 집합이고, 프로퍼티는 `key` 와 `value` 의 쌍이다. 값으로 함수가 오면 그것을 특별히 `메소드` 라고 부른다.

```jsx
const person = {
  name: "희재", // 프로퍼티 (key: 'name', value: '희재')
  greet: function () {
    // 메서드
    console.log(`안녕하세요, ${this.name}입니다.`);
  },
};
```

## 객체 리터럴로 객체 생성

자바스크립트에서는 다양한 객체 생성 방식을 지원한다.

- 객체 리터럴
- Object 생성자 함수
- 생성자 함수
- `Object.create` 메서드
- 클래스

가장 간단한 것은 `{}` 를 사용하는 객체 리터럴이다. 객체 리터럴은 0개 이상의 프로퍼티를 정의한다. 런타임에 평가되어 객체를 생성한다.

## 프로퍼티

### 프로퍼티 접근

```jsx
// 마침표 표기법
person.name; // '다솜'

// 대괄호 표기법 — 키가 변수이거나 식별자 규칙에 어긋날 때 사용
person["name"]; // '다솜'
person["first-name"]; // 하이픈 포함 키는 대괄호만 가능
```

### 프로퍼티 동적 생성

```jsx
const person = {
  name: "하늘",
};

person.age = 30;

console.log(person); // { name: "하늘", age: 30 }
```

### 프로퍼티 삭제

```jsx
delete person.age;

delete person.address;
// person 객체에 address 프로퍼티가 존재하지는 않아 삭제할 수는 없지만 오류 대신 무시한다.
```

## ES6 객체 리터럴 단축 문법

```jsx
const name = "하늘";
const age = 30;

// 프로퍼티 축약 표현 — 변수명과 키 이름이 같으면 생략 가능
const person = { name, age };
// == { name: name, age: age }

// 메서드 축약 표현
const obj = {
  greet() {
    // function 키워드 생략
    console.log("hi");
  },
};

// 계산된 프로퍼티 이름 (Computed property name)
const key = "name";
const obj2 = {
  [key]: "하늘", // { name: '하늘' }
};
```
